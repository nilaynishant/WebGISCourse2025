Introduction

This class introduces web mapping using Leaflet.js from the ground up. It is ideal for GIS practitioners who have little or no programming or web development experience. The course will cover foundational programming concepts for web mapping, the use of Leaflet.js libraries for spatial visualization, integrating geospatial APIs, and building interactive and visually engaging web maps.

Get the Code and Data Package

The code examples in this class use several datasets for creating web maps and adding interactivity. All required datasets, JavaScript files, and HTML templates are provided in the webmapping_leaflet.zip file.

To get started:

1. Download the webmapping_leaflet.zip file.

2. Unzip the file to a directory—preferably to <home folder>/Downloads/webmapping_leaflet/.

The package includes:

• Sample datasets for web mapping (GeoJSON, CSV, etc.).

• Preconfigured HTML templates for creating web maps with Leaflet.

• JavaScript files demonstrating various functionalities like adding markers, popups, layers, and controls.

Module 1: Geoportal Basics

1. Hello World: Creating a Basic Web Map with Leaflet

This tutorial will guide you through the steps to create a basic web map using Leaflet.js. By the end of this tutorial, you will learn how to initialize a map, add tile layers, place markers, draw shapes, and load GeoJSON data.

Step 1: Setting Up the HTML File

To get started, create an HTML file (e.g., index.html) and include the following structure:

<!DOCTYPE html>
<html>
<head>
  <title>Web Mapping Tutorial - Part 1</title>
  <!-- Include Leaflet CSS -->
  <link rel="stylesheet" href="https://unpkg.com/leaflet@1.7.1/dist/leaflet.css" />
  <style>
    #map {
      height: 100vh; /* Set the map height to cover the full viewport */
    }
  </style>
</head>
<body>
  <h1>Web Mapping Tutorial - Part 1</h1>
  <div id="map"></div>

  <!-- Include Leaflet JS -->
  <script src="https://unpkg.com/leaflet@1.7.1/dist/leaflet.js"></script>
  <script>
    // Your JavaScript code will go here
  </script>
</body>
</html>

Step 2: Initializing the Map

Inside the <script> tags, initialize the map with a center point and zoom level using the following code:

var map = L.map('map').setView([25.505, 92.05], 8);

Step 3: Adding a Tile Layer

Add a tile layer to display the map background using OpenStreetMap tiles:

L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
attribution: 'Map data &copy; <a href="https://www.openstreetmap.org/">OpenStreetMap</a> contributors'
}).addTo(map);

2. Adding Layers to a Web Map

This tutorial will guide you through the process of adding Web Map Service (WMS) layers to a Leaflet map. You will add:

  1. A WMS layer from your local GeoServer.
  1. A WMS layer from the NeSDR Geoportal.
  1. GeoJSON file

Step 1: Setting Up the Base Map

The base map is initialized using OpenStreetMap tiles. The following code initializes the map and sets a center point and zoom level:

var map = L.map('map').setView([25.505, 92.05], 8);
L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
attribution: 'Map data &copy; <a href="https://www.openstreetmap.org/">OpenStreetMap</a> contributors'
}).addTo(map);

Step 2: Adding a Marker

Place a marker on the map with a popup:

var marker = L.marker([25.505, 92.05]).addTo(map);
marker.bindPopup('<b>Welcome to the Web Mapping Tutorial!</b><br>This is your starting point.').openPopup();

Step 3: Adding a Local GeoServer WMS Layer

To add a layer from your local GeoServer instance, use the L.tileLayer.wms method. The following example uses the webgisWS:NL_DISTRICT_BOUNDARY_1575 layer:

var wmsLayerUrl = 'http://10.0.40.17:8080/geoserver/webgisWS/wms';
var wmsLayer = L.tileLayer.wms(wmsLayerUrl, {
layers: 'webgisWS:NL_DISTRICT_BOUNDARY_1575', // Specify the layer name
format: 'image/png',
transparent: true,
}).addTo(map);

Step 4: Adding Another Layer from Local GeoServer

To include an additional WMS layer from your GeoServer, update the wmsLayerUrl and specify the desired layer. For example:

// Load GeoJSON from an external file
    fetch('your-geojson-file.geojson') // Replace with the URL to your GeoJSON file
      .then(response => response.json())
      .then(data => {
        // Add GeoJSON layer to the map
        L.geoJSON(data, {
          onEachFeature: function (feature, layer) {
            if (feature.properties && feature.properties.name) {
              layer.bindPopup("<b>" + feature.properties.name + "</b>");
            }
          },
          style: function(feature) {
            return { color: 'blue', weight: 2 };
          }
        }).addTo(map);
      })
      .catch(error => console.error('Error loading GeoJSON:', error));

Step 5: Adding a Layer from NeSDR Geoportal

To include a layer from the NeSDR Geoportal, replace the WMS URL and layer name with the corresponding details. For example:

var nesdrWmsLayerUrl = 'https://nesdr.nesac.gov.in/geoserver/ows';
var nesdrWmsLayer = L.tileLayer.wms(nesdrWmsLayerUrl, {
layers: 'nesdr:LAYER_NAME', // Replace with the NeSDR layer name
format: 'image/png',
transparent: true,
}).addTo(map);

Module 2: Leaflet Control and Plugins

3. Adding Layer Control in Leaflet

In this tutorial, you will learn how to use Leaflet.js to manage and toggle different layers on your web map using the layer control feature. Layer control is an interactive tool that allows users to switch between layers and visualize different types of data on the map.

Key Concepts in Layer Control

  1. Base Layers: These are the background layers of your map (e.g., OpenStreetMap or satellite imagery). Only one base layer can be active at a time.
  1. Overlay Layers: These are additional layers like markers, WMS layers, or GeoJSON data that can be toggled on and off independently.

Step-by-Step Explanation of the Code

1. Initialising the Map

The map is initialised with the base layer using OpenStreetMap tiles:

var map = L.map('map').setView([25.505, 92.05], 8);
var openStreetMapLayer = L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
attribution: 'Map data &copy; <a href="https://www.openstreetmap.org/">OpenStreetMap</a> contributors'
});
openStreetMapLayer.addTo(map);

3. Adding a Marker and WMS layer

A marker is added to the map to represent a specific location:

var marker = L.marker([25.505, 92.05]);
marker.bindPopup("<b>Hello!</b><br>This is a custom marker.").openPopup();
marker.addTo(map);

var wmsLayerUrl = 'https://www.nesdr.gov.in/igistile/NER_Land_Use_50K_QC_passWS/wms';
var wmsLayer = L.tileLayer.wms(wmsLayerUrl, {
layers: 'NER_LULC_50K_1st_CYCLE',
format: 'image/png',
transparent: true,
attribution: 'WMS data &copy; <a href="URL_TO_YOUR_DATA_SOURCE">Data Source</a>'
});

4. Grouping Overlay Layers

The marker and WMS layer are grouped as overlay layers:

var overlayLayers = {
"Marker": marker,
"WMS Layer": wmsLayer
};

5. Adding the Layer Control

The L.control.layers method is used to add the layer control to the map. This allows toggling the overlay layers:

L.control.layers(null, overlayLayers).addTo(map);

• null: Indicates no base layer switching (only one base layer is present).

• overlayLayers: Specifies the layers that can be toggled.

4. Using Plugins in Leaflet - Adding Measurement Tools

Leaflet’s plugin ecosystem provides a powerful way to extend the core functionality of the library. In this tutorial, you’ll learn how to integrate the Leaflet.Measure plugin to add measurement tools for length and area to your web map. Additionally, you’ll customize the measurement units to fit your needs.

What are Plugins?

Plugins in Leaflet are third-party extensions that add additional features or tools, such as measurement tools, clustering, or heatmaps, which are not included in the core Leaflet library. Plugins are easy to integrate and provide enhanced functionality for specialized mapping tasks.

Step-by-Step Guide to Using the Measurement Plugin

1. Include the Plugin Files

To use the Leaflet.Measure plugin, you need to include its CSS and JavaScript files in your HTML file. This is done by adding the following lines in the <head> section:

<link rel="stylesheet" href="https://unpkg.com/leaflet-measure/dist/leaflet-measure.css" />
<script src="https://unpkg.com/leaflet-measure/dist/leaflet-measure.js"></script>

2. Initialise the Map

Set up a basic Leaflet map with a base layer:

var map = L.map('map').setView([25.505, 92.05], 8);
var openStreetMapLayer = L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
attribution: 'Map data &copy; <a href="https://www.openstreetmap.org/">OpenStreetMap</a> contributors'
});
openStreetMapLayer.addTo(map);

3. Add the Measurement Tool

Add the measurement tool to the map using the L.control.measure method. Customize the measurement units for length and area as follows:

L.control.measure({
primaryLengthUnit: 'meters', // Main unit for length measurements
secondaryLengthUnit: 'kilometers', // Secondary unit for length
primaryAreaUnit: 'sqmeters', // Main unit for area measurements
secondaryAreaUnit: 'hectares' // Secondary unit for area
}).addTo(map);

Customizing Measurement Units

The Leaflet.Measure plugin supports the following units for customization:

• Length Units: meters, kilometers, feet, miles, nauticalmiles, yards.

• Area Units: sqmeters, acres, hectares, sqmiles.

Update the primaryLengthUnit, secondaryLengthUnit, primaryAreaUnit, and secondaryAreaUnit parameters as needed. For example:

L.control.measure({
primaryLengthUnit: 'feet',
secondaryLengthUnit: 'miles',
primaryAreaUnit: 'acres',
secondaryAreaUnit: 'sqmiles'
}).addTo(map);

Module 3: API Interaction

5.Implementing a Search Functionality with OpenStreetMap API

In this tutorial, you’ll learn how to add a search feature to your Leaflet web map using the OpenStreetMap (OSM) Nominatim API. This feature allows users to search for locations by name, retrieve their geographical coordinates, and visualize them on the map with a marker and popup.

What is the OpenStreetMap Nominatim API?

The Nominatim API is a free geocoding service provided by OpenStreetMap. It enables you to:

• Forward Geocoding: Convert location names or addresses into latitude and longitude.

• Reverse Geocoding: Convert latitude and longitude into a human-readable address.

In this example, we use forward geocoding to find a location based on a user’s query.

How the Search Feature Works

1. The user enters a location name in the search box.

2. The Nominatim API processes the query and returns a list of possible matches with their coordinates.

3. The map zooms to the selected location and displays a marker with a popup showing the name.

Step-by-Step Guide

1. Adding a Search Box

The search box is implemented using a simple <input> field and a <button>:

<div id="search-container">
<input type="text" id="search" placeholder="Search for a location..." />
<button id="search-btn">Search</button>`
</div>

• Styling: The #search-container is styled with CSS to position it above the map:

#search-container {
top: 10px;
left: 50px;
z-index: 1000;
background: white;
padding: 10px;
border-radius: 5px;
}

#search {
width: 300px;
padding: 5px;
font-size: 14px;
}

#search-btn {
padding: 5px 10px;
font-size: 14px;
cursor: pointer;
}

2. Fetching Results from the Nominatim API

The Nominatim API is queried using JavaScript fetch. The query URL is formatted as:

https://nominatim.openstreetmap.org/search?format=json&q=LOCATION_QUERY

Here’s the search functionality in the script:

document.getElementById('search-btn').addEventListener('click', function () {
var query = document.getElementById('search').value;
if (!query) {
alert('Please enter a location to search.');
return;
}

var url = `https://nominatim.openstreetmap.org/search?format=json&q=${encodeURIComponent(query)}`;
fetch(url)
.then(response => response.json())
.then(data => {
if (data.length > 0) {
var result = data[0];
var lat = parseFloat(result.lat);
var lon = parseFloat(result.lon);

map.setView([lat, lon], 12);
var searchMarker = L.marker([lat, lon]).addTo(map);
searchMarker.bindPopup(`<b>${result.display_name}</b>`).openPopup();
} else {
alert('No results found. Please try another location.');
}
})

.catch(error => {
console.error('Error fetching data from Nominatim API:', error);
alert('An error occurred while searching. Please try again later.');
});
});

6. Map with Buffer Search

This tutorial demonstrates how to integrate a form-based interface with a Leaflet map. The interface allows users to add markers, define a search radius, and fetch nearby features using the Overpass API. The form provides an easy-to-use control for input and interaction with the map.

Key Features

1. Interactive Form Controls:

• Button to enable marker placement.

• Input fields for displaying coordinates and specifying a buffer radius.

• Submit button to execute the buffer search.

2. Marker Placement: Users can click on the map to place a marker.

3. Buffer Visualization: A circle is drawn around the marker to represent the search area.

4. Data Retrieval: Points of interest (POIs) within the buffer are fetched and displayed.

Step-by-Step Guide

1. Setting Up the Map

The map is initialized with a base layer using OpenStreetMap tiles:

var map = L.map('map').setView([25.505, 92.05], 8);

L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
  attribution: 'Map data &copy; <a href="https://www.openstreetmap.org/">OpenStreetMap</a> contributors'
}).addTo(map);

2. Adding the Form Controls

The form interface includes:

• A button to activate marker placement.

• A read-only text field to display the selected coordinates.

• A numeric input field for defining the buffer radius.

• A submit button to execute the search.

<div id="controls">
<button id="add-marker-btn">Add Marker</button>
<label for="coordinates">Coordinates:</label>
<input type="text" id="coordinates" placeholder="Lat, Lng" readonly />
<label for="distance">Distance (meters):</label>
<input type="number" id="distance" placeholder="1000" />
<button id="submit-btn">Submit</button>
</div>

• Styling: The form is styled for a clean and accessible design:

#controls {
margin: 10px;
}

#controls input {
margin: 5px;
}

3. Integrating the Form with the Map

• Activate Marker Placement: Clicking the “Add Marker” button enables a mode to place a marker on the map.

• Display Coordinates: The map click event updates the coordinates in the text input field.

document.getElementById('add-marker-btn').addEventListener('click', function () {
addMarkerMode = true;
alert('Click on the map to add a marker.');
});

map.on('click', function (e) {
if (addMarkerMode) {
var lat = e.latlng.lat.toFixed(6);
var lng = e.latlng.lng.toFixed(6);
// Add or update the marker

if (marker) {
map.removeLayer(marker);
}

marker = L.marker([lat, lng]).addTo(map);

// Update the coordinates in the form

document.getElementById('coordinates').value = `${lat}, ${lng}`;
addMarkerMode = false;
}
});

4. Submitting the Buffer Query

The “Submit” button initiates the buffer search and data retrieval:

• Retrieves the coordinates and radius from the form.

• Draws a circle (buffer) around the marker.

• Fetches nearby features using the Overpass API.

document.getElementById('submit-btn').addEventListener('click', function () {
var coordinates = document.getElementById('coordinates').value;
var distance = document.getElementById('distance').value;
if (!coordinates || !distance) {
alert('Please add a marker and specify a distance.');
return;
}

var [lat, lng] = coordinates.split(',').map(Number);

// Draw buffer circle

if (bufferCircle) {
map.removeLayer(bufferCircle);
}

bufferCircle = L.circle([lat, lng], {
color: 'red',
fillOpacity: 0,
radius: distance
}).addTo(map);

// Fetch nearby features using Overpass API

var query = `
[out:json];
node(around:${distance}, ${lat}, ${lng});
out body;
`;

fetch('https://overpass-api.de/api/interpreter?data=' + encodeURIComponent(query))
.then(response => response.json())
.then(data => {

data.elements.forEach(element => {
if (element.lat && element.lon) {
L.circle([element.lat, element.lon], {
color: 'blue',
fillColor: '#30c',
fillOpacity: 0.5,
radius: 5
})

.bindPopup(`Type: ${element.tags?.amenity || 'Unknown'}<br>Node ID: ${element.id}`)
.addTo(map);
}
});
})

.catch(error => {
console.error('Error fetching data from Overpass API:', error);
});
});
